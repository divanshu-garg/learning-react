## Solutions:

### What is `NPM`?

***
npm is a package manager which is kind of a repository contaaining all the javascript related packages. we can install those packages locally using npm as well. it does not stand for "node package manager" although that is what it does
***

### What is `Parcel/Webpack`? Why do we need it?

***
Parcel is a bundler that helps to make our react code fast, efficient and ready to be served to the end user
***

### What is `.parcel-cache`

***
it is the cache directory that parcel builds(caching files) to run the code very fastly after the first build during which this dir is generated
***

### What is `npx` ?

***
npx is used to execute the javascript packages we installed using npm
***

### What is difference between `dependencies` vs `devDependencies`

***
devDependencies are only used while the project is in the development phase and not pushed to the production while, while the former comes handy for both use cases
***

### What is Tree Shaking?

***
removing unused code during build is called tree shaking. we have all the code stored with us but the build file used to run the project becomes more efficient by doing this
***

### What is Hot Module Replacement?

***
Real time code updation without needing a reload
***

### List down your favourite 5 superpowers of Parcel and describe any 3 of them in your
own words.

***
1) Tree Shaking
2) Hot Module Replacement
3) Local Server
4) Caching
5) Dev Build

laast 3 kind of self explanatory :)
***

### What is `.gitignore`? What should we add and not add into it?

***
the files that can be generated by a script and not to be pushed to git are added to this .gitignore file
***

### What is the difference between `package.json` and `package-lock.json`

***
latter manages the specific version used in development which should also be pushed to production
***

### Why should I not modify `package-lock.json`?

***
it is automatically generated by npm or yarn and there is no need for manual editing. if I make a change in it, most likely will cause some errors in my project
***

### What is `node_modules` ? Is it a good idea to push that on git?

***
it contains all the dependencies & transitive dependencies. it can be regenerated so should not be pushed to git
***

### What is the `dist` folder?

***
dist folder is the folder in which the react code coverted to simple html css and js is kept. this folder is run to see our final output
***

### What is `browserlists`

***
a package that helps to extend support and compatibility of our website to older browsers and versions
***
